# Система учета картриджей (PHP)

Этот каталог содержит full-stack приложение на PHP: API и встроенный веб-интерфейс.

## Что реализовано

- REST API (маршруты через /api)
- JWT-аутентификация (Bearer token)
- Роли доступа (admin/editor/viewer)
- CRUD для пользователей, регионов, картриджей, работ и заметок
- Логи изменений статусов картриджей
- Автоматическая миграция и сидирование при первом запуске
- Пользователь по умолчанию: admin / admin123
- Полностью переработанный темный современный интерфейс
- Многостраничный frontend (главная, создание картриджа, регионы, карточка/история)
- Автокомплит названий при создании картриджа
- Автогенерация номера картриджа (+1 от последнего, с учетом региона и формата)
- Умный поиск и фильтрация картриджей по регионам
- История по картриджу: работы, даты, примечания, смены статусов
- Экспорт в Excel (несколько листов: картриджи, работы, статусы, регионы)

## Стек

- PHP 8.2+
- Microsoft SQL Server (PDO SQLSRV)
- Без внешнего фреймворка (легковесный роутер)

## Что нужно для запуска

1. Установленный SQL Server локально (например SQL Server Developer / Express).
2. В PHP должны быть включены расширения:
- pdo_sqlsrv
- sqlsrv
3. Драйвер ODBC для SQL Server (обычно ODBC Driver 18).

## Запуск локально

1. Скопируйте окружение:

```bash
cd php
cp .env.example .env
```

2. Запустите сервер:

```bash
php -S 0.0.0.0:3001 -t public
```

3. Убедитесь, что в `.env` корректные параметры MSSQL:

```dotenv
DB_HOST=127.0.0.1
DB_PORT=1433
DB_DATABASE=prn_cartridge_db
DB_USERNAME=sa
DB_PASSWORD=YourStrong@Password123
```

Важно: если в `DB_DATABASE` не указан префикс `prn_`, приложение добавит его автоматически.

4. Создайте пустую БД в SQL Server (если еще не создана), например:

```sql
CREATE DATABASE prn_cartridge_db;
```

5. Перезапустите сервер и откройте приложение:

- http://localhost:3001

6. Базовый URL API:

- http://localhost:3001/api

## Данные для входа администратора

- Логин: admin
- Пароль: admin123

## Примечания

- При первом запуске автоматически создаются таблицы и админ-пользователь по умолчанию в MSSQL.
- Встроенный фронтенд обслуживается тем же PHP-сервером и использует API по /api.
- Проект запускается локально без Docker.
- Для создания регионов нужна роль admin.
